# Dream Skill — 文件型记忆库的"睡眠整理"

> 模仿人脑做梦 5 件事：**巩固 · 关联 · 稳态 · 冲洗 · 髓鞘化**。把散落的日志/笔记整合成结构化长期记忆，去重瘦身，清理垃圾，把高频回路固化成技能。
>
> 大模型不会自动从错误中学习——Dream = 人工模拟"睡眠学习"。

## 前置：三层架构（自动检测 + 缺失 onboarding）

本 skill 面向三层 markdown 记忆库。**Phase 0 会自动检测**——动态定位项目目录（`$CLAUDE_PROJECT_DIR`/cwd，**不写死路径，兼容任何用户**），检测三层结构是否存在：
- ✅ 已搭好 → 正常跑 dream
- ❌ 缺失 → **onboarding**：mkdir 三层目录 + 写骨架模板（MEMORY/CONVENTIONS/ARCHITECTURE）+ 给 CC 机制指引（触发词 / SessionEnd hook 存 transcripts，**只指引不改用户配置**）

搭建细节见 [`references/memory-setup.md`](references/memory-setup.md)。

## 触发词

做梦、dream、复盘、整理记忆、睡眠学习、巩固知识、清理记忆库、记忆库维护

隐性触发：记忆库久未整理、日志堆积、知识条目零散、索引膨胀

## 红线（不可逾越）

- ⛔ **原始层（transcripts/逐字稿）只读**——永不删/移/改。等价于生物不删感官原始登记；它是 ground truth，删除=永久丢失
- **禁 `rm`**，删除一律 `mv` 到 `trash/日期/`
- 过时条目只标 `stale` 不真删
- **保守修剪**（≠生物激进修剪）：信息丢失代价不对称，只做标记/归档/列待删清单，**高危删除等用户确认**
- **先实测再下结论**：别拿过期数据 / 错误路径当现状（核实路径、看 mtime）
- **只整理不创造**：dream 只整理人已写入的内容（去重/关联/瘦身/清理/固化），**不创造新知识、不编造事实**；提炼可派弱模型，但"写什么/删什么/进哪类"的判定主线程把关

## 6 阶段（仿睡眠周期）

| Phase | 做什么 | 详解 |
|---|---|---|
| 0 入睡校准 | 读总揽(索引+规范+蓝图) → ls 计数 → 关键文件 shasum 快照(回滚锚) → 定范围 | phases §0 |
| 1 慢波巩固 | 读近期日志 → 提炼到知识层 → 重复事件合并成规律 | phases §1 |
| 2 REM 关联 | 跨条目找隐藏关联 → 补 wiki-link → 生新想法 → 找孤儿条目 | phases §2 |
| 3 稳态瘦身 | 去重合并 → 索引瘦身(<150行) → 过时标 stale | phases §3 |
| 4 类淋巴冲洗 | 清磁盘垃圾(缓存/僵尸/旧产物) → ⛔不碰原始层/脚本 → mv trash | phases §4 |
| 5 髓鞘化+自检 | 高频流程固化 skill → 校验回路 → 项目归档检测 → 架构自检(实际 vs 蓝图) | phases §5 |
| 6 清醒回忆 | 输出 diff 摘要 → 高危动作列清单等确认 → 追加日志 | phases §6 |

完整流程见 [`references/dream-phases.md`](references/dream-phases.md)。

## 执行纪律

- **先总揽后局部**——不全量 find/grep 探查，先读索引/规范/蓝图
- **动条目前 grep 其引用链**——避免删出悬空链接
- **提炼可派弱模型**（省强模型算力），但**判定主线程把关** + 真跑验证
- **幂等可回滚**——dream 前快照，流程跑两次结果一致
- **不搞 cron**——手动触发（机械定时可能无料可巩固，靠用户触发更可靠）

## 偶尔提醒（不 cron，你说才跑）

dream **不自动定时跑**。但为避免记忆库长期不整理而腐化：
- **新会话自检**：CC 开新会话时，若发现 `log/` 有近期未提炼的重要内容，**主动建议**用户跑 dream
- **只提醒不代跑**：建议后等用户说"做梦"才执行——控制权在你
- 模式：**偶尔提醒（不漏）+ 你说才跑（不乱）**

## 何时跳过 / 轻量跑

- 近期已 dream 过 → 提示用户，跳过或只跑变化的
- 库太小（日志 <3 篇）→ 没料可巩固，提示
- 纯架构/回路问题 → 跳过 Phase 1-2，直奔 Phase 5
