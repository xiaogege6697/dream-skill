# Dream Skill

文件型记忆库的"睡眠整理"工作流——模仿人脑做梦，对 markdown 知识库做**巩固 · 关联 · 瘦身 · 清理 · 固化回路**。

## 为什么需要

大模型有四大结构缺陷：**会话隔离 / 上下文有限 / 不自动学习 / 无元认知**。文件型记忆库 + Dream 工作流 = 用文件模拟长期记忆、索引模拟回忆、**Dream 模拟睡眠学习**，弥补这四缺陷。

## 快速开始

1. **搭架构**（首次）：按 [`references/memory-setup.md`](references/memory-setup.md) 搭三层记忆库（约 10 分钟）
2. **日常**：工作时往日志层追加摘要；会话逐字稿自动落原始层
3. **做梦**：说"做梦 / dream / 复盘"触发本 skill，跑 6 阶段整理

## 核心理念

- **手写精选 > 自动流水账**——只记值得长期留的
- **三层分离**——日志(14天归档) / 知识(永久) / 原始(只增不减)，互不污染
- **归档优于删除**——永不 rm，过时标记，保留可回溯
- **保守修剪**——信息丢失代价不对称，宁多留不误删

## 生物学映射

| 人脑睡眠功能 | Dream 动作 |
|---|---|
| 慢波巩固（海马→皮层 replay）| 日志提炼到知识层 |
| REM 关联（远距皮层连接）| 补 wiki-link、生新想法 |
| 突触稳态（renormalization）| 去重瘦身 |
| 类淋巴冲洗（冲代谢废物）| 清磁盘垃圾 |
| 髓鞘化（强回路加固）| 高频流程固化成 skill |

**关键偏离**：人脑激进修剪（一晚削 ~20% 突触），本工作流**保守修剪**——信息丢失代价不对称，宁可多留。
